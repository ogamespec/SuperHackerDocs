                                      FV
              Функция вычисляет будущее значение после серии оди-
              наковых платежей.
              
Синтаксис     FV(<число1>,<число2>,<число3>)
              
              Число1 представляет величину периодического  плате-
              жа. Число2  представляет процентную ставку на пери-
              од, выраженную  в виде десятичного числа (например,
              .12 представляет  12% ставку).  Число3 представляет
              количество периодов.
              
Описание      FV иногда называют будущим  или  сложным  значением
              ренты, потому  что вы можете использовать эту функ-
              цию для вычисления величины, накапливающейся в рен-
              те при осуществлении периодических платежей.
              
Примеры       Вычислить суммарную  величину при ежегодных взносах
              в 109.53 доллара за 48 лет  при  начислении  каждый
              год 12.5%.
              
              MESSAGE FV(109.53,.125,48)

              вильным и полным значением для этого поля.
              
Синтаксис     ISVALID()
              
Описание      Выдает True,  если текущее содержимое поля является
              правильным и полным значением для  этого  поля,   в
              противном случае False.  Если система Paradox нахо-
              дится не в режимах редактирования (EDIT)  или  сов-
              местного редактирования (COEDIT), то данная функция
              всегда выдает True.

                                   LINKTYPE
              
              Выдает тип связи между таблицами  в  многотабличной
              форме.
              
Синтаксис     LINKTYPE()
              
Описание      Выдает символьную строку, содержащую тип связи меж-
              ду главной и текущей подчиненной таблицей в  много-
              табличной форме. Это может быть одно из значений:
              - "1-1 Group" - связь один-к-одному;
              - "1-M Group" - связь один-ко-многим;
              - "Group"     -     связи     многие-к-одному     и
              многие-ко-многим;
              - "None" - если текущей является  главная  таблица,
              нет связи   между таблицами или нет активной много-
              табличной формы.
              Тип связи  представляется на экране с помощью инди-
              катора (записи типа 1-1 Group) в верхнем правом уг-
              лу экрана. Если на рабочей области нет образов, или
              текущий образ не является таблицей,  выдается ошиб-
              ка.
              
Примеры       Функцию LINKTYPE можно использовать  для  определе-
              ния, какую форму команды WAIT применять в многотаб-
              личной форме.
              IF LINKTYPE()="None"
                  THEN
                    WAIT RECORD
               PROMPT "F4 подчиненные записи,F2- выход"
                    UNTIL "F2","F4"
                  ELSE
                    WAIT TABLE
               PROMPT "F3 главная запись,F2- выход"
                     UNTIL "F2","F3"
              ENDIF

См.также      LOG для вычисления логарифма числа по основанию 10.

                                     MATCH
              
              Проверяет, соответствует  ли данная строка образцу.
              
Синтаксис     MATCH(<строка>,<образец>,<переменная 1>,...,
              <переменная n>)
              
Описание      Строка-образец, состоит из  произвольных  символов,
              чередующихся с элементами универсального сравнения,
              такими, как ".." и "@". Элемент ".." сопоставляется
              с любым   количеством символов,  в то время как "@"
              сопоставляется только с единственным символом.  Как
              и в запросах,  различие верхнего и нижнего регистра
              не происходит.  Если сравнение прошло успешно,   то
              выдается значение True, в противном случае - False.
              Если функция снабжена именами переменных  (присутс-
              твуют поля <переменная 1>,  <переменная n>),  и со-
              поставление прошло успешно,  то части строки сопос-
              тавляются с элементами универсального сопоставления
              и присваиваются переменным слева направо.
              
              t=MATCH("Dogfood","D..")      ; t=True
              
              t=MATCH("Dogfood","dog..")    ; t=True
              
              t=MATCH("Dogfood","..d..d..") ; t=True
              
              t=MATCH("Dogfood","..d..d..",x,y,z)
              ; x=" ",y="ogfoo", z=" ", t=True

Описание      Параметр <степень  округления>  должно  быть  целым

Примеры       MESSAGE ROUND (123.456,0)     ; =123
