					      Дмитрий Кохманюк

		НЕКОТОРЫЕ СВЕДЕНИЯ О DR-DOS 5.0

DR-DOS, DR-DOS ... Стоит ли? Судите сами.

Любители работать в командной строке Command.com, это для вас!
Вам больше не нужна программа dosedit -- вставьте в Config.Sys:
HISTORY = ON|OFF [,nnn[, ON|OFF]]
(Смысл параметров: включить режим запоминания команд; размер
буфера; включить режим вставки по умолчанию.)

Вообще язык Config.Sys значительно расширен. Судите сами:

REM text
ECHO text
Между прочим, ECHO переводит все в большие буквы -- сервис! :-)

CHAIN filename
Переключиться на интерпретацию другого файла.

:label
GOTO  label
Ясно.

? "prompt [y/n] "  command
Выполнение команды после ввода "Y"

FASTOPEN = nnn
Аналогично программе PC-DOS.

INSTALL   = filename [parameters]
Аналогично DOS 4.0.

HIDOS = ON
Максимально выгружать ядро DOS в high memory.
Необходима также загрузка драйвера HIDOS.Sys.

HIINSTALL = filename [parameters]
Как INSTALL, но с загрузкой в верхние адреса памяти (если возможно)

HIDEVICE = filename [parameters]
Соответствует (HIINSTALL - INSTALL) + DEVICE (guess yourself ...)

Кроме того, если в корневом каталоге имеется файл DConfig.Sys,
выполняется он и только он.

Теперь о совместимости.

Пользователи ADM будут удивлены увеличением числа логических дисков.
DR-DOS поддерживает разделы ADM самостоятельно (разумеется, защита не
обеспечивается). Будут видны первые 4 раздела независимо от прав
доступа.

Не функционируют некоторые вирусы (ах, какая жалость!).

Популярный драйвер Config.Ctl отказывается работать.
Проблему можно решить примерно следующим способом:

?"Change system configuration? [Y/N]: " install=editor  startup.sys
chain startup.sys

Обратите внимание на ДВА пробела в команде INSTALL -- второй
из них идет на корм синтаксическому анализатору ...

Boot-сектор просто великолепен:  нет никаких ограничений на системные
файлы, они могут располагаться в любом месте диска, *не* быть
непрерывными, а их имена в корневом каталоге могут находиться где
угодно.

Свободно поддерживаются нестандартные форматы гибких дисков, включая
10- и 18-секторные (400K, 800K, 1500K и т.д.)

Команды assign, subst, more встроены в Command.Com.  Вследствие этого
пользователям 4DOS придется писать aliasы вроде:

alias subst command.com /c subst %&

Таким образом, размер ядра составляет соответственно 15K, 25K и 52K
(в PC-DOS 37K).

При использовании драйвера HIDOS.Sys на машине без отображаемой памяти
(беэ 80386 и без C&T RAM (Neat Chip set)) для выгрузки DOS теряется вся
HMA (High Memory Area), т.е. первые 64K extended memory. Фактически, мы
платим 64K extended memory за 22K обычной.

У DR-DOS еще много неоткрытых тайн. Буду весьма благодарен за любую
дополнительную информацию.
